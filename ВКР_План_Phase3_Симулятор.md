## 2. Выбор стенда (день 1)

См. `03_Симулятор/_выбор_стенда.md` (создаётся первой задачей фазы).

**Шортлист:**

| Стенд | Плюсы | Минусы | Сложность мат-модели |
|---|---|---|---|
| S1 — Термокамера со стабилизацией PT100 | Простая физическая модель (теплоперенос ODE 1-го порядка), классика метрологии, наглядно для 3D | Не самый «впечатляющий» | низкая |
| S2 — SCPI-мультиметр DC + источник | Промышленная типичность, реалистичные тайминги settling, прямой выход на SCPI/VISA | Сложнее визуализация | средняя |
| S3 — Газовый хроматограф (упрощённая модель) | Эффектно для презентации | Нужен серьёзный домен (хроматография), сложно валидировать | высокая |
| S4 — Виброиспытательный стенд | Эффектно, есть открытые модели sin-sweep | Сложная DSP-часть | средняя |

**Решение по умолчанию:** **S1 + S2 двойка**. S1 — основной (для 3D и наглядности), S2 — второй для статистики и обобщаемости. Если время поджимает — S1 only.

## 4. Физическая модель S1 (детализация)

**Уравнение теплопереноса 1-го порядка:**

dT/dt = (T_set − T) / τ + ξ(t),

где τ — постоянная времени системы, ξ(t) — гауссовский шум с σ ≈ 0.05 °C.

**Сенсор PT100:** R(T) = R₀ · (1 + α·T + β·T²), R₀ = 100 Ом, α = 3.9083e-3, β = -5.775e-7 (по ГОСТ 6651-2009).

**АЦП:** 16-бит, диапазон 0–500 °C, квантование ≈ 0.0076 °C. Дрейф нуля 1e-3 °C/час.

**Параметры по умолчанию:**
- T_set = 150 °C
- τ = 60 с
- T_min = 50 °C (порог HEAT → HOLD)
- δ_stable = 0.02 °C/с (порог стабилизации)
- t_hold_min = 300 с
- N_min = 20

## 6. 3D-визуализация (вариант)

**Минимально приличный вариант (1-2 дня):** HTML + Three.js, простая 3D-сцена:
- Куб (термокамера) с прозрачными стенками.
- Цилиндр (образец) меняет цвет от синего к красному в зависимости от T.
- Линия температуры (live chart рядом).
- Подсветка состояния FSM (INIT/HEAT/HOLD/MEASURE/POST) текстом сверху.
- Запись прогона из JSONL и проигрывание со слайдером времени.

**План B (если Three.js «съест» время):** matplotlib-анимация (FuncAnimation) + одна 3D-картинка из pyvista для иллюстрации в ПЗ. Делается за день.

## 7. Критерии завершения

- Эталонный прогон S1 без нарушений: event-log соответствует ожидаемому FSM-пути.
- Каждый из 7 типов инъекций воспроизводимо появляется в логе.
- 3D-визуализация показывает: образец, термопару, диаграмму T(t), активное состояние FSM.
- Симулятор детерминирован при фиксированном seed.
- Юнит-тесты зелёные (`pytest 03_Симулятор/tests`).
