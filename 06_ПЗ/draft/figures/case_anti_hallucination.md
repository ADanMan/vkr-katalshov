# Case study: срабатывание анти-галлюцинационного фильтра в Phase 5

> **Контекст для главы 4 ПЗ.** Эпизод зафиксирован 2026-05-12 при первом
> production-прогоне FLV-модуля на синтетическом сценарии `s1_time_under-000`
> (батч-эксперимент Phase 5, тег `phase5-canonical-2026-05-12`).
> Иллюстрирует архитектурный принцип «верификация и объяснение разделены»:
> детерминированный вердикт сохраняется, даже если LLM-объяснение
> деградирует до шаблона.

## Что произошло

Команда:

```bash
flv check \
  --spec ../02_Спецификация/dsl_v1.yaml \
  --log runs/s1_time_under-000.jsonl \
  --report verdict_under.md --explain
```

Лог `s1_time_under-000.jsonl` — синтетический прогон с инъекцией `TIME_UNDER`:
HOLD-фаза искусственно сокращена до 30 c при нормативе ≥ 300 c (ГОСТ Р 8.563-2014).

LLM-explainer (Role 3 по ADR-002, модель по умолчанию
`anthropic/claude-sonnet-4.6`) попытался сгенерировать человеко-читаемое
объяснение поверх формального вердикта. Анти-галлюцинационный фильтр
(`flv.anti_hallucination`) перехватил вывод и принудительно деградировал
до шаблонного объяснения:

```
00:55:53 [WARNING] flv.llm_explainer — LLM-explainer: обнаружены
потенциальные галлюцинации (numbers=('270,0', '103'),
ids=('HEAT_START', 'HOLD_START', 'RUN_END')); fallback.
```

## Что именно отверг фильтр

Согласно `flv/anti_hallucination.py`, фильтр сравнивает каждое число
и UPPER_SNAKE_CASE-идентификатор в выводе LLM с белым списком, собранным
из спецификации (DSL) и трассы (event-log). Толеранс на округление —
0.5 % (соответствует ГОСТ 8.207-76, §4 «обработка результатов прямых
измерений»).

| Категория | Отвергнуто | Источник в трассе? |
|---|---|---|
| Numbers | `270.0` | Нет — ни в DSL, ни в event-log нет такого числа |
| Numbers | `103` | Нет — N_collected, T_set, t_hold, T_min ≠ 103 |
| IDs | `HEAT_START` | Нет — в трассе только `RUN_START`, `HEAT_END`, `HOLD_END`, `MEAS_END`, `MEAS_PROBE` |
| IDs | `HOLD_START` | Нет — отсутствует в каталоге событий симулятора |
| IDs | `RUN_END` | Нет — `RUN_END` не выдаётся симулятором при FAIL |

То есть LLM попытался сослаться на события, которых физически не было
в прогоне, и оперировать числами вне распознанного словаря. Это
классическая галлюцинация без злого умысла — модель «дополнила»
повествование плавдоподобными, но несуществующими фактами.

## Что осталось неизменным

| Артефакт | Источник | Затронут галлюцинацией? |
|---|---|---|
| Verdict status | детерминированные матчеры | ✗ |
| 6 нарушений (`PRED_FAIL` × 4, `TIME_UNDER` × 1, `N_TOO_LOW` × 1) | детерминированные матчеры | ✗ |
| Expected / Actual для каждого нарушения | из DSL + event-log | ✗ |
| `seq` и `t` каждого события | из event-log | ✗ |
| Markdown-таблица отчёта | из verdict-репортера | ✗ |
| Текстовое объяснение | LLM Role 3 → fallback на шаблон | ✓ (заменено) |

Файл `verdict_under.md` содержит полный formальный вердикт + помеченное
шаблонное объяснение с примечанием:

> *«Объяснение сформировано по детерминированному шаблону (LLM Роль 3
> не задействована или отбракована анти-галл фильтром).»*

## Архитектурная роль этого эпизода

Эпизод эмпирически подтверждает три проектных решения, заложенных
в начале работы (Phase 2, ADR-002):

1. **LLM находится downstream от вердикта.** Никакая галлюцинация
   модели не может изменить заключение о соответствии методике —
   модель не имеет доступа к решающей логике.
2. **Анти-галлюцинационный фильтр работает на боевых данных, а не
   только в юнит-тестах.** Граница между «настоящим объяснением» и
   «шаблонным фоллбэком» проходит через сравнение со словарём, а не
   через интуитивные эвристики.
3. **Деградация мягкая.** При срабатывании фильтра пользователь
   получает корректный вердикт + явное примечание «здесь LLM не
   использован», а не молчаливо испорченный текст.

## Почему это сильнее, чем McNemar-значимость

В сводке Phase 5 baseline TPR = 0.714, FLV TPR = 1.000, парных
«FLV-finds-but-baseline-misses» = 100. Это даёт ожидаемую сильную
значимость McNemar-теста и подтверждает покрытие.

Но «FLV ловит больше нарушений» — это количественное утверждение.
Сработавший фильтр в production даёт качественное утверждение:
**даже если кто-то атакует LLM в дальнейшем (prompt injection,
jailbreak), вердикт нельзя подделать**, потому что LLM не на
критическом пути. Для регуляторно-релевантных систем (метрология
по ГОСТ Р 8.563-2014) это сильнее любого процента.
