## Концепция работы

В ИИС нормативная методика измерения формулируется в текстовом виде (ГОСТ/МИ), а исполняется как программный сценарий. Между ними возникает **семантический разрыв**: код может быть синтаксически корректным, но не соответствовать методике (например, `wait(30)` вместо `wait(300)`).

Метод FLV закрывает этот разрыв: формализует методику в DSL → автомат → сравнивает с фактической трассой исполнения сценария → выдаёт вердикт OK/FAIL с детализацией нарушений (пропуск шага, недовыдержка, неудовлетворённый предикат стабилизации, выход из диапазона).

Работа реализует прототип FLV-модуля на Python и демонстрирует его на симуляторе термокамеры (PT100 + АЦП), сопоставляя метрики baseline (точечные ad-hoc проверки) и proposed (FLV).
