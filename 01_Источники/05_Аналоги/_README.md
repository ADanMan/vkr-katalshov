# 05_Аналоги — сравнительный обзор инструментов верификации

> Phase 1 Catalog. Дата формирования: 2026-05-03.

## Назначение папки

Здесь хранится сравнительный обзор инструментов, которые либо претендуют на ту же нишу, что и наш FLV-метод, либо являются ближайшими родственниками по подходу
(static analysis для test sequences, runtime verification по STL/MTL, model checking по timed automata, requirements traceability).

Обзор используется для:
- Главы 1 ПЗ, раздел 1.3 «Существующие подходы к верификации программных моделей измерительных процессов».
- Обоснования научной новизны и инженерной ниши работы (см. итоговые выводы в `_сравнение.md`).

## Структура

- `_сравнение.md` — главный файл: пер-инструментные карточки, сводная таблица, выводы, ссылки.
- `_README.md` — этот файл.

## Принцип ведения

- Источники — только официальные сайты, GitHub репозитории, рецензируемые статьи.
- Никаких pdf здесь не лежит — нам нужна сводка, не первичные тексты. Если потребуется конкретная статья (например, Donze 2010 или Nickovic 2020 RTAMT), она докачивается в `01_Источники/04_Литература/01_FLV_RV/`.
- Каждый инструмент описан по одной шаблонной анкете (см. `_сравнение.md`), что позволяет автоматически собирать таблицы для ПЗ.
- Цитаты из официальной документации — не более 1-2 строк, в кавычках, со ссылкой на URL.

## Категории

| Категория | Инструменты | Зачем нам |
|---|---|---|
| A | NI TestStand Sequence Analyzer, NI VI Analyzer, NI Requirements Gateway, NI LabVIEW Unit Test Framework | Прямой контекст: реальные инструменты разработки ИИС в LabVIEW/TestStand. Показывают, что в индустрии нет инструмента уровня FLV. |
| Б | RTAMT, Breach, py-metric-temporal-logic, TeLEx, MonPoly | Формальный фундамент (STL/MTL/MFOTL). Показывают, что технология есть, но без DSL для нормативных методик и без сопоставления с трассой ИИС. |
| В | UPPAAL | Контекст для главы про timed automata; не прямой аналог, но академический baseline. |
