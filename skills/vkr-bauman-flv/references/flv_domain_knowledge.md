# Доменные знания по FLV — ключевые работы и термины

## Терминология

- **Информационно-измерительная система (ИИС)** — комплекс средств измерений, автоматизированно собирающий, обрабатывающий и представляющий измерительную информацию. ГОСТ Р 8.596-2002.
- **Методика измерений (МИ)** — совокупность операций и правил, выполнение которых обеспечивает получение результата с известной погрешностью. ГОСТ Р 8.563-2014.
- **Измерительный протокол** — программная реализация методики измерений в ИИС.
- **Семантический разрыв** — расхождение между нормативной (текстовой) методикой и её программной реализацией; синтаксис кода корректен, но смысл нарушен.
- **Функционально-логическая верификация (FLV)** — наш термин: сопоставление фактической трассы исполнения программного протокола с формализованной моделью методики (FSM/Timed FSM + предикаты + временные окна).
- **Runtime Verification (RV)** — общая область, к которой относится FLV. Цель — проверка свойств системы по ходу её исполнения.
- **Signal Temporal Logic (STL)** — формализм для описания свойств непрерывных сигналов с количественной семантикой.
- **Metric Temporal Logic (MTL)** — обобщение LTL с временными ограничениями.
- **Timed Automaton** — конечный автомат с действительными часами (Alur, Dill 1994).

## Существующие инструменты-аналоги

| Инструмент | Что делает | Что НЕ делает (наша ниша) |
|---|---|---|
| NI TestStand Sequence Analyzer | Статический анализ TestStand-сценариев на стиль и corner cases. | Не сопоставляет фактическую трассу с нормативной моделью. |
| NI VI Analyzer Toolkit | Проверка стиля/правил кода LabVIEW VI. | Не верифицирует поведение. |
| NI Requirements Gateway | Трассируемость требований ↔ артефакты. | Не проверяет фактическое исполнение. |
| RTAMT (Python) | Online STL-мониторинг сигналов. | Без интеграции в контур ИИС, без DSL для методики. |
| Breach (MATLAB) | Параметрический STL-анализ. | Закрытый стек MATLAB. |
| py-metric-temporal-logic | Python-парсер MTL. | Нет offline-/online-комбо, нет интеграции с реальным measurement-стеком. |

**Наша ниша:** связка «нормативная методика → формальная DSL-спецификация → Timed FSM → matcher против фактической трассы исполнения программного сценария ИИС». Это сквозной workflow, которого нет ни у одного из перечисленных.
