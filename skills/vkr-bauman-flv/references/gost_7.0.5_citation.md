# ГОСТ 7.0.5-2008 — библиографические ссылки. Шпаргалка

Применяется для оформления списка использованных источников в ВКР. Это форма представления, **отличающаяся** от ГОСТ 7.1-2003 (который описывает библиографическую запись для каталогов).

## Общие правила

- Каждая ссылка — отдельный пункт, сквозная нумерация по работе в порядке первого упоминания.
- Размер шрифта — TNR 14 pt (как у основного текста), межстрочный 1.5.
- Нумерация — арабскими, с точкой и пробелом: `1. `, `2. `.
- Между фамилиями авторов — запятая. Инициалы — после фамилии (а не до!): «Иванов И.И.», «Петров П.П.». Между авторами — запятая, перед последним — без союза «и».
- Если авторов > 4 — пишется «и др.» после первых трёх (для русских).
- Английская транслитерация авторов — допустима для международных источников.

## Знаки препинания

ГОСТ 7.0.5 предписывает специальные «области описания»:

```
Авторы. Заглавие : сведения, относящиеся к заглавию / Сведения об ответственности. — Сведения об издании. — Место издания : Издательство, Год. — Объём.
```

- Двоеточие (`:`) — перед сведениями, относящимися к заглавию (подзаголовок).
- Косая черта (`/`) — перед сведениями об ответственности (например, под редакцией).
- Точка с тире (`. — `) — между областями описания.
- Двоеточие в выходных данных — между местом и издательством: «М. : Наука».

## Стиль для смешанных источников

- Иностранные источники — на языке оригинала. Транслитерацию авторов — только если в работе все источники переводятся.
- Кириллические источники — без транслитерации.
- Издательство — `М. : Наука` (русское), `Cham : Springer` (для международного).
- Год — через запятую перед годом издания.

## Workflow в нашем проекте

1. Каждый источник попадает в `BIBLIO.bib` сразу при скачивании, с уникальным ключом `авторгод_шорт`.
2. При написании главы в Markdown — пометка `[@bibkey]` (стиль pandoc).
3. При финальной сборке `.docx` — pandoc + CSL-стиль `gost-r-7-0-5-2008.csl` либо ручная сборка списка по шаблонам выше.
4. Перед сдачей — `scripts/validate_bibtex_gost.py` проверяет полноту полей.

## CSL для pandoc

CSL-стиль ГОСТ 7.0.5 доступен на <https://citationstyles.org/> и <https://github.com/citation-style-language/styles>. Файл — `gost-r-7-0-5-2008.csl` (или `russian-gost-r-7-0-5-2008-numeric.csl`). Кладём в `assets/`, подключаем флагом `--csl assets/gost-r-7-0-5-2008.csl`.

## Типовые ошибки оформления библиографии

- ✗ APA-стиль вместо ГОСТ («Maler, O., & Nickovic, D. (2004). Monitoring...»).
- ✗ Курсив на названии книги (ГОСТ — без курсива; курсив только если редакция требует).
- ✗ Инициалы перед фамилией («И.И. Иванов» вместо «Иванов И.И.»).
- ✗ Отсутствие даты обращения у электронных источников.
- ✗ Знак тире вместо длинного — без пробелов («1991-384» вместо «1991. — 384 с.»).
- ✗ Единый список без сортировки в порядке упоминания (если требуется — алфавит, но обычно — по упоминанию).
