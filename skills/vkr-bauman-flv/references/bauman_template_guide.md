- `01_Источники/03_МГТУ/polozhenie_vkrb_K4.pdf` — основное Положение о ВКРБ МГТУ им. Н.Э. Баумана (Учёный совет, 12.10.2015), 4.79 МБ.

- Реферат
- Содержание
- Перечень сокращений (заготовка)

Используй этот бланк как стартовый `.docx`-файл при финальной сборке: открой его в Word, удали лишний placeholder-текст из К4, вставь свои разделы из Markdown-черновиков через pandoc.

### 3. Стили из бланка

Бланк уже настроен на:
- TNR 14 для основного текста.

- Поля 30/10/20/20 мм.
- 1.5 межстрочный.

Это совпадает с требованиями ГОСТ 7.32. Не сбивай эти стили при копи-пасте из Markdown.

## Что брать из Положения о ВКРБ (`polozhenie_vkrb_K4.pdf`)

Документ объёмный (4.8 МБ, ~50 стр.), читается сетево по нужным разделам. Ключевое:

### Виды работ

ВКР бакалавра — самостоятельная научно-исследовательская/инженерная работа. У нас — НИР с инженерной реализацией.

### Объём работы

- Бакалаврская ВКР: 50-70 стр. основного текста (без приложений).
- Иллюстраций — 10-20.
- Источников — 20-40.

Наша цель: **65-75 страниц основного текста**, ≥ 25 источников. Внутри лимитов.

### Сроки

- Защита бакалавра — июнь.
- Допуск к защите — за 1-2 недели до защиты, с отзывом руководителя и оценкой нормоконтроля.
- Антиплагиат — обычно ≥ 65% оригинальности (МГТУ требование).

### Защита

- Доклад 7-10 минут.
- Презентация 12-15 слайдов.
- Вопросы комиссии — 5-10 минут.
- Демонстрация работы прототипа — приветствуется (у нас есть симулятор + FLV).

### Отзыв руководителя

Готовится руководителем за 1-2 недели до защиты. Шаблон отзыва — в приложениях к Положению.

### Рецензия

Для бакалавров — обычно не требуется (только для специалистов и магистров), но кафедра К2 может попросить. Уточни у Поярков Н.Г./Комарова Е.Г.

## Что брать из Приложения к Положению (`polozhenie_vkrb_pril_K4.pdf`)

Содержит формы:
- Заявление на тему ВКР.
- Задание на ВКР.
- Календарный план выполнения ВКР.
- Отзыв руководителя.
- Лист нормоконтроля.

Если у Данилы нет утверждённого Задания на ВКР — это первый блокер. См. `99_Артефакты/blockers.md`.

## Workflow финальной сборки `.docx`

```
1. Открой 10_blank_vkrb_K4.doc как baseline (Save As → ВКР_Катальшов.docx).
2. Замени реквизиты титульного листа на К2 МФ МГТУ.
3. Удали placeholder-текст в каждом разделе.
4. Сконвертируй каждую Markdown-главу из 06_ПЗ/draft/ в DOCX через pandoc + reference.docx (см. scripts/md_to_docx_gost.py).
5. Импортируй DOCX-куски в основной файл через "Вставка → Объект → Текст из файла" в Word.
6. Сгенерируй "Содержание" автоматически (Word: References → Table of Contents).
7. Проверь нумерацию страниц, заголовки, подписи.
8. Прогон через антиплагиат.
```
