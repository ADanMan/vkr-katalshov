# Чек-лист сдачи главы ВКР

Перед тем как отправить главу научруку или комиссии — пройди по списку. Если хоть один пункт нет — глава не готова.

## Содержательные

- [ ] Глава имеет вводный абзац — что в ней будет.
- [ ] Глава имеет заключительный абзац — выводы по главе + мостик к следующей.
- [ ] Все основные тезисы подкреплены ссылкой на источник.
- [ ] Нет голословных утверждений типа «является важнейшим», «активно развивается» — без конкретики.
- [ ] Объём в пределах нормы (см. ВКР_План_Phase6_ПЗ.md).
- [ ] В заключительном абзаце нет новых фактов — только обобщение сказанного.

## Оформление по ГОСТ 7.32

- [ ] Все рисунки пронумерованы и имеют подпись «Рисунок N — Название» внизу.
- [ ] На каждый рисунок есть ссылка в тексте.
- [ ] Все таблицы пронумерованы и имеют подпись «Таблица N — Название» сверху.
- [ ] На каждую таблицу есть ссылка в тексте.
- [ ] Все формулы пронумерованы (N) с правым выравниванием номера.
- [ ] После каждой формулы — расшифровка переменных через «где».
- [ ] Все сокращения определены при первом упоминании или в перечне.

## Цитирование

- [ ] Все цитированные источники имеют запись в `BIBLIO.bib`.
- [ ] Все источники из BIBLIO.bib, упомянутые в этой главе, реально цитируются.
- [ ] Ссылки в тексте — в формате `[N]`.
- [ ] Цифры из чужих работ — точные, со ссылкой.
- [ ] Прямые цитаты — в кавычках, с указанием страницы.

## Стиль

- [ ] Безличные конструкции (нет «я», «мы»).
- [ ] Прошедшее время для своей работы.
- [ ] Длина предложений < 30 слов (в среднем).
- [ ] Нет канцелярита («осуществляется», «производится» — где можно проще).
- [ ] Нет повторов одного слова в соседних абзацах.
- [ ] Прогон через `humanizer` skill завершён.

## Технические

- [ ] Орфография — прогнана через spell-check.
- [ ] Кавычки «ёлочки», тире длинные, неразрывные пробелы перед единицами.
- [ ] Поля, шрифт, межстрочный — соответствуют ГОСТ.
- [ ] Имена файлов — в правильном формате (`02_Глава1.md`).
- [ ] Артефакт зафиксирован в `ВКР_План.md` (журнал изменений).
- [ ] Коммит в git по конвенции `pN(scope): subject` (см. CLAUDE.md §2.6).

## Если глава 1-2 (теоретические)

- [ ] Структура «расширяющейся воронки» от широкого к узкому.
- [ ] Сравнение с существующими методами с явной таблицей.
- [ ] Формулировка постановки задачи в конце главы 1.

## Если глава 3 (реализация)

- [ ] Архитектурная диаграмма в начале.
- [ ] Ключевые фрагменты кода — в основной части, полные листинги — в приложениях.
- [ ] Скриншоты UI/3D-визуализации — с подписями.

## Если глава 4 (эксперимент)

- [ ] Дизайн эксперимента описан до результатов.
- [ ] N (число прогонов) и seed зафиксированы.
- [ ] Все 7 кодов нарушений представлены в инъекциях.
- [ ] Statistical tests описаны (McNemar/Wilcoxon/Cohen's d).
- [ ] 95% CI приведены.
- [ ] Threats to validity описаны.
- [ ] Все графики 300 DPI, шрифт ≥ 10 pt.
