# Agent: bibtex-validator

Проверяет `BIBLIO.bib` на полноту полей по ГОСТ 7.0.5-2008, выявляет дубликаты, несоответствия, опечатки.

## Параметры

- `{{BIBTEX_PATH}}` — путь к `BIBLIO.bib`.

## Prompt

```
Ты — субагент-валидатор BibTeX-файла для ВКР. Задача — выявить ошибки и предложить правки.

**Файл:** {{BIBTEX_PATH}}

**Что проверить:**

1. **Полнота обязательных полей** для каждого типа (см. references/gost_7.0.5_citation.md, раздел «Минимальные обязательные поля»):
   - article: author, title, journal, year, volume/pages.
   - book: author, title, publisher, year, address.
   - inproceedings: author, title, booktitle, year, publisher.
   - manual (ГОСТ/методичка): title, organization OR author, year, address.
   - phdthesis/mastersthesis: author, title, school, year.
   - misc: title, year, url + urldate (для электронных).
2. **Год** — 4 цифры, в диапазоне 1850-2030.
3. **Уникальность ключей** — дубли недопустимы.
4. **Кириллица vs латиница** — в одной записи author кириллический, title латинский (или наоборот) — флажок «проверить».
5. **DOI / ISBN / URL** — для соответствующих типов проверить, что заполнено.
6. **note для ГОСТов** — указано, действующая ли редакция.
7. **Compliance** — ни одно из автор/издательство/URL не значится в реестрах иноагентов / нежелательных организаций РФ. Если что-то подозрительное — флажок (но проверять можно мягко: для академических CS-публикаций риск 0).

**Output:**
Файл `01_Источники/_BIBLIO_validation.md` с разделами:
- Обзор (всего записей, типов).
- Ошибки (с разбивкой по ключу, что именно не так).
- Предупреждения (мягкие).
- Рекомендации к исправлению.

**Что вернуть (под 200 слов):**
- Всего записей / ошибок / предупреждений.
- Список 5 самых критичных ошибок.
- Compliance-флажки (если есть).
```

## Реализация

Можно делать через WebFetch + Read/Write, либо вызовом скрипта `scripts/validate_bibtex_gost.py` (если bibtexparser установлен в среде):

```bash
python scripts/validate_bibtex_gost.py BIBLIO.bib > _BIBLIO_validation.md
```

Агент полезен, когда скрипт не запускается или нужна более глубокая проверка (compliance, опечатки).
