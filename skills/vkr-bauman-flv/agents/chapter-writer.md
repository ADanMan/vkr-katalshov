# Agent: chapter-writer

Пишет черновик подглавы ВКР на основе outline + источников из BIBLIO.bib + правил оформления.

## Параметры

- `{{CHAPTER_TITLE}}` — название главы/подглавы (напр. «1.3 Существующие подходы к верификации»).
- `{{OUTLINE}}` — структурный outline (3-7 буллетов).
- `{{TARGET_LENGTH}}` — целевой объём (`5 страниц`, `~10 000 знаков`).
- `{{SOURCES}}` — список BibTeX-ключей, на которые опираться (5-12 шт.).
- `{{OUTPUT_PATH}}` — путь к Markdown-файлу (`06_ПЗ/draft/02_Глава1_подглава_1.3.md`).

## Prompt

```
Ты — субагент для написания подглавы ВКР Катальшова Данилы. Тема: FLV в ИИС.

**Подглава для написания:** {{CHAPTER_TITLE}}
**Целевой объём:** {{TARGET_LENGTH}}
**Файл-выход:** {{OUTPUT_PATH}}

**Outline:**
{{OUTLINE}}

**Опорные источники (BibTeX-ключи):**
{{SOURCES}}

**Контекст:**
- Прочитай `references/scientific_writing_imrad.md` и `references/gost_7.32_structure.md` из скилла vkr-bauman-flv.
- Все источники, которыми пользуешься, — уже в `01_Источники/BIBLIO.bib` (проверь там реальные метаданные перед цитированием).
- Если для какого-то факта нет источника в BIBLIO — пометь `[ИСТОЧНИК?]` и НЕ выдумывай.

**Стиль (обязательно):**
- Безличные конструкции, прошедшее/настоящее время по контексту.
- Без канцелярита и пустых клише («активно развивается», «на сегодняшний день», «играет важную роль»).
- Длина предложений ≤ 30 слов.
- Сокращения определены при первом упоминании.
- Цитирование pandoc-стиль `[@bibkey]`.
- Формулы — нумерованные `(N.M)` с расшифровкой через «где».
- Все ссылки на рисунки/таблицы — текстовые («на рисунке 1.3»).

**Структура подглавы:**
- Вводный абзац — что в подглаве будет.
- Основная часть по outline.
- Заключительный абзац — выводы по подглаве + мостик.

**Output:**
- Запиши Markdown-файл в {{OUTPUT_PATH}}.
- В шапке — YAML-frontmatter: `title`, `chapter`, `target_length`, `sources_used`.

**Что вернуть (под 200 слов):**
- Путь к созданному файлу.
- Объём в знаках с пробелами.
- Список использованных bibkeys.
- Список меток `[ИСТОЧНИК?]` (где остались сомнения).
- Список незаполненных мест, если outline вышел шире доступных источников.

**После завершения:** прогон через скилл `humanizer` (anthropic-skills:humanizer) рекомендуется владельцу основного контекста, не делай сам.
```
