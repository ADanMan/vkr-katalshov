# ADR-001 — Инструменты эмуляции МК и диаграмм

> Дата: 2026-05-03. Статус: principal direction. Подлежит уточнению на старте Phase 3.

## Контекст

Тема ВКР — FLV измерительных протоколов в составе ИИС. Phase 3 предусматривает симулятор стенда S1 (термокамера со стабилизацией PT100). Изначально был принят чистый Python-симулятор как primary track.

Возникла идея демонстрировать FLV не только над Python-абстракцией, но и над **реально исполняемым кодом микроконтроллера**, чтобы показать переносимость метода на embedded-сцены. Также возник вопрос про более экспрессивные диаграммы (Mermaid → Excalidraw).

## Решения

### 1. Симулятор стенда — двухуровневый

**Primary (обязательный):** Python-симулятор (`03_Симулятор/sim/`). Полностью контролируемый, детерминированный (seed), статистически воспроизводимый. Используется для всех 100+ прогонов Phase 5.

**Demo-track (если успеваем):** микроконтроллерный эмулятор Wokwi или Velxio. Один Arduino-скетч на C++, реализующий ту же FSM-методику. Serial-вывод формат event-log JSONL — те же ключи событий, что у Python-симулятора. FLV-модуль проверяет лог одинаково независимо от источника.

**Выбор эмулятора (по предпочтительности):**

| # | Инструмент | Плюсы | Минусы | Решение |
|---|---|---|---|---|
| 1 | **Wokwi** (wokwi.com) | Зрелый проект (5+ лет), цитируется в академических работах, есть CLI и VS Code extension, бесплатные публичные проекты, поддержка ESP32 / AVR / RP2040 | SaaS (без офлайн), для приватных проектов нужен Wokwi Pro | **рекомендован** для финальной демонстрации на защите |
| 2 | **Velxio** (velxio.dev) | Open-source, self-hostable через docker-compose, можно накрутить кастомный MCP, контроль на 100% | Молодой проект, меньше комьюнити, риск багов | **резерв** + хорош для self-hosted lab |
| 3 | SimulIDE | Desktop, AVR/ARM, анимация схем | Менее интегрируется с современным Python-стеком | для случаев, если SaaS недоступен |
| 4 | Proteus VSM | Промышленный стандарт | Платная лицензия, **только Windows** | резерв на отдельной машине; Данила может поставить на Windows-комп для контрольной проверки, на Mac нет (актуальный девайс — Mac) |

**По умолчанию демо-трек:** Wokwi для презентации (узнаваемость + надёжность), Velxio как self-hosted альтернатива в репо для воспроизводимости.

**Если бюджета времени нет — отказаться от demo-track**, остаётся Python-only. Это не отменяет валидность работы.

### 2. MCP для эмулятора

Если делаем demo-track, есть две опции:
1. **Pre-recorded run.** Запустить Wokwi/Velxio один раз, сохранить serial-лог как fixture в репо, FLV-модуль проверяет fixture. Простой путь.
2. **Live MCP integration.** Поднять локально Velxio (docker-compose), накрутить MCP-обёртку на его API, я смогу запускать прогоны и читать serial. Сложно, риск увеличения объёма работ на 1-2 недели.

**Рекомендация:** Pre-recorded run для ВКР; live MCP — после защиты, если будет публикация.

### 3. Диаграммы — троекратный набор

**Mermaid** (через `scripts/mermaid_to_png.py`):
- Архитектурные блок-схемы — глава 2-3.
- FSM измерительного процесса (`stateDiagram-v2`).
- Sequence-диаграммы online-режима.
- Gantt-план ВКР.

**Excalidraw** (через GitHub-skill, см. ниже):
- Эскизные схемы для слайдов защиты (hand-drawn стиль).
- Архитектурные комбо для презентации (более запоминаются комиссии).
- **Для ПЗ — тоже допустимо** (по решению Данилы 2026-05-03): где Mermaid выглядит слишком "схематично" и нужна выразительность визуала, можно вставить Excalidraw-PNG. Главное — единство стиля внутри одной главы.

**schemdraw** (Python lib):
- Электрические принципиальные схемы (PT100 4-wire, схема измерительной цепи).
- Программная генерация = git-friendly.

**Velxio/Wokwi** (если идём в demo-track):
- Скриншот эмулятора в ПЗ как иллюстрация рабочего MCU-кода.

### 4. Excalidraw — какой скилл взять

Шортлист GitHub-скиллов (с проверкой по числу звёзд / активности):
- **`coleam00/excalidraw-diagram-skill`** — Playwright-render pipeline, агент видит свой output и self-corrects. Лучший выбор для автоматического качества.
- **`robonuggets/excalidraw-skill`** — 10 техник + MCP для self-correcting.
- **`yctimlin/mcp_excalidraw`** — full canvas toolkit через MCP.

**Рекомендация:** `coleam00/excalidraw-diagram-skill` как baseline, при желании — установить MCP `yctimlin/mcp_excalidraw` для live-canvas.

## Последствия

- Phase 3 раздваивается на Python-track (обязательный, primary) и MCU-track (опциональный, demo).
- Объём работы Phase 3 не меняется существенно: MCU-track делается 1-2 дня поверх готовой методики.
- Объём ПЗ растёт незначительно: добавится 1-2 страницы про переносимость метода в embedded.
- Появляется привлекательность для презентации: «работает не только в Python, но и на реальном Arduino».
