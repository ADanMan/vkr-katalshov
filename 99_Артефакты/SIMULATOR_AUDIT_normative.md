## 2. Что сделано корректно

### 2.1 PT100 R(T) — ГОСТ 6651-2009, формула Каллендар-Ван Дюссена

В `sim/config.py`:
```
R0 = 100 Ом
α = 3.9083 × 10⁻³ K⁻¹
β = -5.775 × 10⁻⁷ K⁻²
```

Проверочный расчёт:
```
W₁₀₀ = R(100°C) / R(0°C) = (1 + α·100 + β·100²) / 1
     = 1 + 0.39083 - 0.005775
     = 1.385055
```

Это ровно нормативное значение W₁₀₀ = 1.385 для платины класса A/B по
ГОСТ 6651-2009 п. 5.1.1 (соответствует МЭК 60751 c TCR α₀ = 0.00385 °C⁻¹).

### 2.2 Допуски PT100 — ГОСТ 6651-2009 табл. A.1

В `sim/uncertainty_model.py` функция `pt100_class_tolerance_C()`:
- Класс A: ±(0.15 + 0.002·|T|) °C ✓
- Класс B: ±(0.30 + 0.005·|T|) °C ✓

### 2.3 СКО результата измерения — ГОСТ 8.207-76 п. 2.4

```
S(A) = √[ Σ(xᵢ - A)² / (n(n-1)) ] = s / √n
```

В `estimate_uncertainty_from_samples()` использовано
`samples.std(ddof=1) / sqrt(n)` — точно эта формула.

### 2.4 Композиция систематических — ГОСТ 8.207-76 п. 4.2-4.3

При отсутствии данных о виде распределения принимается равномерное:
δ → u = δ / √3.

В коде: `pt100_tol / sqrt(3)`, `q / sqrt(12)`, `drift / sqrt(3)` — всё в
соответствии п. 4.2.

### 2.5 Квантование АЦП — равномерное распределение

`u_quant = q / √12`. Стандартное выражение по GUM B.2.2; согласовано с
ГОСТ 8.009-84.

### 2.6 Структура методики — ГОСТ Р 8.563-2009

FSM `INIT → HEAT → HOLD → MEASURE → POST` соответствует типовой структуре
методики термоиспытаний:
- INIT — подготовка средства измерений;
- HEAT — установление режима;
- HOLD — выдержка с условием стабилизации;
- MEASURE — съём отсчётов;
- POST — обработка результатов.

Формальное оформление методики (как отдельный документ ГОСТ Р 8.563-2009
форма) — отдельный артефакт; в Phase 3 не делается.

## 3. Выявленные пробелы

### Пробел 3.1 — Коэффициент Стьюдента вместо k=2 (ГОСТ 8.207-76 п. 3.2)

**Что в коде сейчас.** В `estimate_uncertainty_from_samples()` параметр
`coverage_factor=2.0` зашит по умолчанию. Это нормальное приближение для
больших выборок и стандартное GUM-значение.

**Что требует ГОСТ 8.207-76 п. 3.2.** Доверительные границы случайной
погрешности находят по формуле ε = t·S(A), где t — коэффициент Стьюдента
из приложения 2 в зависимости от n и P. При n=20, P=0.95 → t = 2.093 ≠ 2.

**Влияние.** На реальных выборках N_min=20 разница ~5 % от расширенной
неопределённости. Не критично, но формально несоответствие.

**Исправление.** Поддержать опцию `coverage_factor='auto'`, при которой
k берётся как t-Стьюдента через `scipy.stats.t.ppf` с df=n-1. По умолчанию
оставить 2.0 для совместимости с GUM-стилем; явно опционально включать
ГОСТ-стиль.

### Пробел 3.2 — Само-нагрев PT100 (ГОСТ 6651-2009 п. 5.4)

**Что в коде сейчас.** Учитываются: допуск класса, шум АЦП, квантование,
дрейф нуля. Само-нагрева нет.

**Что требует ГОСТ 6651-2009.** Самонагрев δT_self при ИКП (источник
постоянного тока) определяется коэффициентом рассеяния тепла:
```
δT_self = P / (k_рассеяния)
```
где P = I²·R — мощность, выделяющаяся в датчике. Для типового PT100 при
I=1 мА, R=110 Ом → P ≈ 0.11 мВт; типовой k_рассеяния = 5 мВт/К → δT_self ≈
0.022 °C.

**Влияние.** Систематическая составляющая ~0.05 °C; того же порядка, что
квантование АЦП. На итоговую u_combined влияет умеренно.

**Исправление.** Добавить в `config.py` константу
`SENSOR_SELF_HEATING_C` (по умолчанию 0.05 °C) и компонент `u_self_heating`
в бюджет неопределённости (равномерное распределение → u = δ/√3).

### Пробел 3.3 — Явное разделение Δ vs θ vs U (ГОСТ 8.207-76 п. 5)

**Что в коде сейчас.** Все компоненты сразу комбинируются в `u_combined`
через RSS; на выходе одна цифра.

**Что требует ГОСТ 8.207-76 п. 5.** При суммировании случайной (Δ) и
систематической (θ) погрешностей нужно сравнивать их соотношение:
- если θ/S(A) < 0.8 → можно пренебречь систематической;
- если θ/S(A) > 8 → можно пренебречь случайной;
- иначе — комбинированная по специальной формуле через коэффициент K
  (раздел 5.2 ГОСТ).

**Влияние.** Скорее методическое: для нашего стенда S(A) ~ 0.01 °C
(шум объекта в HOLD), θ ~ 0.5 °C (PT100 класс A при T=150 °C). Соотношение
θ/S(A) ≈ 50 — то есть случайной составляющей можно пренебречь и итоговая
погрешность определяется θ. Но в коде это не зафиксировано.

**Исправление.** Добавить в `UncertaintyBudget.to_dict()` поле
`dominant_component` с указанием, какая категория преобладает; для
формального соответствия ГОСТ — функция `gost8207_combine(samples,
random_components, systematic_components)` в дополнение к GUM-композиции.
В рамках Phase 3 ограничиться диагностическим полем; полная функция —
post-defense.

## 6. Что не входит в Phase 3 и зафиксировано как ограничение

- Поверка средств измерений (ГОСТ 8.461-2009, ссылка из ГОСТ 6651-2009).
  Симулятор не имитирует процедуру поверки; работа предполагает уже
  поверенный сенсор и АЦП.
- Полный документ методики измерений в формате ГОСТ Р 8.563-2009 (с
  титульным листом, разделами «Назначение», «Метод измерений» и т.д.).
  Для ВКР этот документ может быть приложением; сейчас методика
  представлена в виде DSL `02_Спецификация/dsl_v1.yaml`, который
  семантически эквивалентен.
- Метрологическая прослеживаемость (ГОСТ Р 8.596-2002 п. 4.3) — отдельная
  тема; в симуляторе не имеет смысла, добавляется на уровне FLV-сертификата
  (Phase 4).

Эти ограничения упомянуть в главе 4 ПЗ как «threats to validity».
