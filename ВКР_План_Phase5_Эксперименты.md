# Phase 5 — Эксперименты и статистика

> Срок: 2026-05-25 → 2026-05-30 (6 дней). Зависимости: Phase 3 (симулятор) и Phase 4 (FLV-модуль) функциональны и интегрируемы.

---

## 1. Цель

Получить численные доказательства эффективности FLV-метода: пара «baseline vs proposed» на двух стендах, статистически значимая разница по детекции нарушений, приемлемый overhead.

---

## 2. Дизайн эксперимента

### 2.1 Стенды и прогоны

| Стенд | Корректные | С инъекциями | Всего |
|---|---|---|---|
| S1 (термокамера) | 50 | 50 (≈ 7 на каждый из 7 кодов нарушения) | 100 |
| S2 (SCPI) | 30 | 30 | 60 (если успеваем) |

Все прогоны параметризованы seed'ом для воспроизводимости. Сценарии — из каталога Phase 3.

### 2.2 Сравниваемые подходы

- **Baseline:** наивные ad-hoc проверки в коде сценария (типа `assert wait_time >= 300`), без формальной спецификации.
- **Proposed (FLV):** прогон через модуль FLV с DSL-спецификацией.

### 2.3 Условия

- Симуляция детерминированная при фиксированном seed.
- Каждый сценарий запускается обоими подходами.
- Прогоны независимы; никаких side-effects между прогонами.

---

## 3. Метрики

### 3.1 Детекция нарушений

| Метрика | Формула | Цель |
|---|---|---|
| TP | детектировал, реально была инъекция | ↑ |
| FP | детектировал, инъекции не было | ↓ |
| TN | не детектировал, инъекции не было | ↑ |
| FN | не детектировал, инъекция была | ↓ |
| TPR | TP / (TP + FN) | ≥ 0.90 |
| TNR | TN / (TN + FP) | ≥ 0.90 |
| FPR | FP / (FP + TN) | ≤ 0.10 |
| FNR | FN / (FN + TP) | ≤ 0.10 |

### 3.2 Качество измерения

- **J_timing** — средняя относительная ошибка по критичным временным окнам (mean(|t_fact − t_spec_min| / t_spec_min)).
- **K_seq** — доля прогонов без нарушения порядка (целевая ≥ 0.95 в proposed).
- **N_reject** (online) — число заблокированных «плохих» прогонов в режиме online.
- **Δbias** — смещение результата измерения (среднее ΔT) при нарушении vs корректно.

### 3.3 Производительность

- **Overhead** — wall-time FLV / wall-time baseline.
- **CPU-overhead** — отношение CPU-time.
- **Memory** — пиковое потребление.

## 4. Статистическая обработка

| Тип данных | Тест | Назначение |
|---|---|---|
| Бинарные исходы (детекция) | McNemar test (paired) | Разница TPR/TNR между подходами |
| Непрерывные (J_timing, overhead) | Wilcoxon signed-rank (paired) | Без предположения о нормальности |
| Непрерывные (нормально распределённые) | Paired t-test | Альтернатива Вилкоксону |
| Все непрерывные | Cohen's d | Размер эффекта |
| Все метрики | Bootstrap 95% CI (10 000 повторений) | Доверительные интервалы |

Уровень значимости α = 0.05.

## 7. Критерии завершения

- ≥ 100 прогонов на S1, ≥ 60 на S2 (если успеем).
- Целевые: TPR ≥ 0.9, FPR ≤ 0.1 (на FLV).
- p-value McNemar < 0.05 для разницы baseline vs FLV.
- Все 7 графиков сделаны и положены в `figures/`.
- Статистический отчёт содержит p-values, CI, эффект-сайз, текстовую интерпретацию.

## 8. Риски

| Риск | Меры |
|---|---|
| Целевые метрики не достигнуты | Калибровка сценариев — увеличиваем «явность» инъекций; разделяем категории «strong» и «subtle» |
| Прогоны медленные | Параллелизация через `multiprocessing`; кэшируем повторяющиеся вычисления |
| Графики не смотрятся | Используем seaborn + matplotlib, единый стиль, единый размер шрифта |
| Не хватает данных для статистики | Увеличиваем n до 200 (легко при детерминированном симуляторе) |
