## 3. Содержание ключевых артефактов

### 3.1 DSL (YAML)

Минимальный набор сущностей: `meta`, `process`, `states[]`, `parameters{}`, `transitions[]`, `checks[]`, `violations_catalog{}`.

Состояния — именованные. Переходы — с гардами (булева логика над переменными), временными ограничениями (`time.min`, `time.max`) и опциональным STL-предикатом стабилизации.

### 3.2 Формальная модель

Timed FSM = (S, S₀, V, P, T), где S — множество состояний, S₀ — начальное, V — переменные процесса, P — предикаты, T ⊆ S × Guard × Time × P × S — переходы. Семантика: исполнение = последовательность пар (состояние, временная метка). Нарушение = трасса не принимается автоматом.

### 3.3 Event-log

JSONL, одна строка = одно событие. Минимальные поля: `ts` (ISO-8601), `run_id`, `state`, `event` (START/END/TICK/ERROR), `params{}`. Опциональные: `signals{}` (для STL).

### 3.4 Архитектура FLV

Компоненты:
- **Spec Loader** — YAML → внутренняя FSM-модель.
- **Log Parser** — JSONL → нормализованная последовательность.
- **Matcher Pipeline** — sequence → timing → predicate.
- **Verdict Aggregator** — OK/FAIL + список нарушений.
- **Reporter** — Markdown/JSON отчёт.
- **Online Adapter** (опционально) — TCP-стрим, инкрементальный матчинг.

### 3.5 Каталог нарушений

| Код | Название | Тип проверки |
|---|---|---|
| SEQ_MISS | Пропуск обязательного шага | sequence |
| SEQ_ORDER | Нарушение порядка переходов | sequence |
| TIME_UNDER | Недостаточная длительность состояния | timing |
| TIME_OVER | Превышение допустимой длительности | timing |
| PRED_FAIL | Неудовлетворённый предикат стабилизации | predicate |
| RANGE_MISM | Параметр вне допустимого диапазона | predicate |
| N_TOO_LOW | Недостаточное число измерений | predicate |

## 4. Критерии завершения

- DSL валидируется собственной JSON Schema (тест `validate.py`).
- Все 7 кодов нарушений формально описаны и для каждого есть пример события-нарушителя.
- Mermaid-диаграмма архитектуры рендерится без ошибок.
- В `formal_model.md` есть нотация и ≥ 1 пример прогона на бумаге.
- Все артефакты лежат в `02_Спецификация/`.

## 5. Риски

| Риск | Меры |
|---|---|
| Сложность тайм-автоматов | Ограничиваемся discrete-time STL (Maler-style), без чистой continuous-time MTL |
| DSL раздут | Жёсткие лимиты: 5 типов состояний, 4 типа гардов, 3 типа предикатов |
| Несовместимость с Phase 4 | До конца Phase 2 — пайротип loader.py, который читает DSL и печатает FSM-граф |
